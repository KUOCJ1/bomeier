# 鉑魅兒 Soft Atelier 整站氛圍重整設計規格

日期：2026-07-11
狀態：待使用者審核
範圍：前台整站視覺、互動節奏、商品呈現與後台視覺一致性

## 1. 設計目標

鉑魅兒目前網站已具備商品、客製、誌與後台管理功能，但整體感受仍偏「一般商品網站加上一些品牌裝飾」。本次重整目標是把網站升級成「輕精品選物工作室」：第一眼要有留白、質感與安靜的設計感；細看時要保留手作、柔光與客製溫度。

本次選定方向為 Soft Atelier：以輕精品選物為主軸，混入手作工作室的柔光與紙感。它不是潮流品牌，也不是過度浪漫的手作文案頁，而是介於精品陳列與溫柔選物之間。

成功標準：

- 首頁第一屏看起來像品牌策展空間，而不是模板式電商首頁。
- 商品卡更像選物標籤，圖片更大、資訊更乾淨、互動更細膩。
- 客製頁像進入工作室選材，而不是填一張表單。
- 誌頁像品牌雜誌，而不是文章清單。
- 後台維持實用，但視覺密度與字體要與品牌前台一致。
- 所有動畫慢、輕、克制，不使用炫技效果。

## 2. 視覺語言

### 色彩

主背景從目前偏奶油色調整為更乾淨的暖白／紙白，避免整站變成單一米色。建議色彩系統：

- Paper White：`#FBF8F4`，整站主背景。
- Warm Porcelain：`#F3ECE4`，區塊背景與柔和分隔。
- Ink Brown：`#1F1915`，主文字，比純黑更溫潤。
- Soft Taupe：`#7A6A5E`，內文與說明文字。
- Rose Gold：`#C9956B`，主要品牌點綴與 CTA。
- Deep Atelier：`#171310`，少量用於高對比按鈕與 footer。
- Mist Blue / Sage / Blush：保留作為五種風格的輕微分類提示，不讓分類色主導整站。

整站避免大量漸層、發光球、過重陰影。光感應來自圖片、留白與細微邊界，不靠裝飾堆疊。

### 字體

中文標題應使用更有精品氣質的明體感，搭配乾淨的無襯線內文。英文品牌字可保留 display serif，但要更克制。

建議：

- 中文標題：`Noto Serif TC`，字重 500 或 600。
- 中文內文與 UI：`Noto Sans TC`，字重 400 或 500。
- 英文 display：`Playfair Display` 或既有 serif display，減少粗體使用。
- UI 小字：提高字距但不過度，維持可讀性。

禁止用過大的英雄字體硬撐質感。主標可以大，但要靠行距、留白與圖片關係建立氣氛。

### 圖片與解析度

商品是賣氛圍與設計感，圖片品質必須成為整站核心。規範如下：

- 首頁與風格卡使用高解析情境圖，避免占位式或低解析圖出現在第一屏與重要區塊。
- 商品卡主圖統一使用 4:5 或 1:1，背景保持一致或至少色溫一致。
- 第二張 hover 圖可作為情境圖，使用慢速推近與淡入，不做快速翻轉。
- 若商品圖解析度不足，卡片不要放太大；反之重要商品可用大圖模組呈現。
- 後台上傳圖片需提示建議尺寸：商品主圖至少 1200px，首頁或風格情境圖至少 1600px。

## 3. 首頁設計

首頁第一屏要從現在的「發光裝飾 hero」改成「柔光工作室場景」。

結構：

- 導覽列維持簡潔，但降低壓迫感，背景半透明紙白，滾動後略微浮起。
- Hero 使用一張高品質商品／工作室氛圍圖作主視覺，文字疊在留白區，不放卡片。
- 主標保留品牌詩意，但減少過度裝飾。
- CTA 維持兩個：先看風格、直接客製。
- 第一屏底部露出下一段商品或風格內容，讓使用者自然往下滾。

建議文案方向：

- H1：日常裡，安靜發光的配飾
- Supporting copy：手作琉璃、金屬配件與色彩搭配，替手機鏈與日常小物留下更細緻的表情。
- CTA：先看風格／開始客製

首頁第二段不應只是「最新商品」四張卡。建議改成策展式區塊：

- 左側一段品牌選物語氣。
- 右側 2 到 4 件代表商品。
- 商品卡資訊簡化，只保留品名、風格、價格。

## 4. 商品頁設計

商品頁的功能已經有搜尋、類型、排序與風格篩選。下一步是把它從工具列感改成選物感。

改造方向：

- 篩選列視覺更輕，像 boutique filter，而不是後台工具。
- 搜尋、類型、排序整合成一條安靜的控制列；手機版改為上下排列。
- 商品卡圖片比例固定，主圖放大，文字區縮小。
- 品名使用較有質感的 serif 或中性 sans，價格降低視覺重量。
- 收藏與購物車按鈕改成低干擾圖示，不要搶商品圖注意力。
- 商品 hover 動畫改成：主圖微推近、情境圖柔淡入、卡片略微浮起。

空狀態文案要更品牌化，例如「這個條件下暫時沒有作品，可以換一種風格看看。」

## 5. 商品詳情頁設計

商品詳情頁要像小型作品頁，不只是 SKU 資訊。

結構：

- 左側大圖與縮圖，圖片框更大、更乾淨。
- 右側商品資訊分層：品名、風格、價格、材質、描述、標籤、CTA。
- 描述文字拉長行距，呈現像選物介紹。
- 狀態標籤降低視覺重量。
- 相關商品使用同一張卡片系統，不再看起來像另一套元件。

## 6. 客製頁設計

客製頁應該是整站最有「工作室」感的頁面。

改造方向：

- 首段改為「選風格、選材質、補需求」的三步節奏。
- 五個風格卡使用更大的情境圖，並放入適合場景、適合穿搭、一眼感受。
- 表單分段顯示，讓使用者不覺得一次填太多。
- 參考圖上傳區改成更像 moodboard drop zone。
- CTA 文案從「送出」改成更有人味，例如「送出客製想法」。

## 7. 誌頁設計

誌頁要像品牌雜誌入口。

改造方向：

- 上方放一篇精選文章大卡，搭配一張高品質圖。
- 其他文章用較安靜的 list/card 混合排版。
- 文章內文預設收合或進入詳情，避免列表頁過重。
- 字體、行距與段落寬度要明顯改善可讀性。

## 8. 關於頁設計

關於頁目前敘事完整但偏長。改為三層：

- 第一段：3 句品牌立場。
- 第二段：品牌時間線或手作流程。
- 第三段：選物原則與客製態度。

視覺上使用更少卡片，更多留白、橫向分隔與局部圖片。

## 9. 後台設計

後台不需要像前台一樣情緒化，但要乾淨、穩定、可長時間使用。

改造方向：

- 字體、按鈕、表格密度與前台保持同一套品牌基礎。
- 商品圖片上傳區加上建議尺寸與格式提醒。
- 商品列表縮圖提高一致性，避免後台看起來像系統雜訊。
- 狀態標籤顏色降低飽和度。
- 操作按鈕更清楚分層：主要、次要、危險。

## 10. 動畫與滾動節奏

動畫原則：慢、輕、可感知，但不表演。

應加入：

- 首屏文字分層淡入，總時長約 900ms 到 1200ms。
- 商品卡進場 stagger，間隔約 60ms。
- 圖片 hover 微推近，scale 不超過 1.04。
- 區塊滾動進場使用 opacity + translateY 16px，不使用大幅滑動。
- 導覽列滾動後變薄、背景更實。
- 尊重 `prefers-reduced-motion`，使用者若偏好減少動態則關閉動畫。

避免：

- 快速翻卡。
- 大幅 parallax。
- 旋轉或發光動畫。
- 過多同時出現的元素。

## 11. 技術實作邊界

本次為既有靜態前台與 Supabase 後台的視覺重整，不重寫架構，不更換框架。

預計改動：

- `public/css/style.css`：新增 Soft Atelier token、首頁、商品卡、頁面區塊與動畫調整。
- `public/js/main.js`：補強 scroll reveal 與 reduced-motion 支援。
- `public/index.html`：調整首頁首屏與策展式商品段落。
- `public/products.html`：控制列與商品頁節奏調整。
- `public/custom.html` 與 `public/js/custom.js`：客製頁視覺節奏與上傳區呈現。
- `public/journal.html` 與 `public/js/journal.js`：誌頁分層呈現。
- `public/css/admin.css`：後台視覺一致性與上傳提示。

不在本次做：

- 不重做整個後台資料模型。
- 不新增付款金流。
- 不新增大型動畫套件。
- 不把商品圖全部重新生成。
- 不更換網站部署方式。

## 12. 驗證標準

完成後至少驗證：

- 首頁桌機與手機第一屏：品牌感、文字可讀、下一段有露出。
- 商品頁：搜尋、排序、分類仍正常。
- 商品卡：圖片不破圖、hover 不突兀、文字不溢出。
- 客製頁：風格卡與表單在手機不擠壓。
- 誌頁：列表不過重，文章可讀。
- 後台：商品圖片上傳按鈕仍可見且不破版。
- 建置通過。
- GitHub Pages 與 ChatGPT Sites 都能更新。

## 13. 風險與處理

最大風險是圖片品質不一致。若程式視覺升級但商品圖解析度不足，質感仍會被拖垮。處理方式是先讓版面更高級，並在後台與商品規格中明確標示建議圖片尺寸；後續再分批替換首屏、風格卡與代表商品圖。

第二個風險是動畫過度。此次只做慢速 reveal、hover 和微互動，不做會干擾購物的動效。

第三個風險是改太多造成既有功能壞掉。實作時要先保留現有資料流與 Supabase 同步邏輯，只改呈現層。
